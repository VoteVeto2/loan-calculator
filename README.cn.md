# 贷款计算器

[English](./README.md) | **简体中文**

一个单页贷款记账工具。录入一笔贷款，设置自定义罚息利率区间，登记分期还款，查看每一笔还款如何在利息和本金之间分摊，并显示实时余额。每次计算都会保存为一个命名的会话（session），方便日后继续查看。

无需构建步骤，运行时除 Google Fonts 外没有任何依赖。直接打开 `index.html` 即可使用。

## 项目结构

```
loan-calculator/
  index.html        仅包含页面结构（标记）
  styles.css        配色 + 布局
  js/
    engine.js       纯贷款计算与格式化（不涉及 DOM、不涉及存储）
    fixtures.js     5 个验收测试场景及预期结果
    store.js        基于 localStorage 的会话持久化
    render.js       视图层：输入数据，输出 DOM
    app.js          控制器：状态、事件、自动保存、启动
  PRD.html          产品需求文档（在浏览器中打开）
  onboard.md        仓库的开发者导览（参与开发请从这里开始）
  test-engine.mjs   无依赖的引擎测试（Node）
  test-dom.mjs      DOM 集成测试（Node + jsdom）
  package.json      声明 jsdom 开发依赖 + 测试脚本
```

浏览器按依赖顺序以传统 `<script>` 标签加载这些 JS 文件（`engine → fixtures → store → render → app`）。它们刻意不使用 ES 模块，因为当页面直接从磁盘（`file://`）打开时，ES 模块的 import 会被 CORS 拦截；而传统脚本可以通过双击直接运行。

## 会话（你的数据会被保存）

每次计算都是一个会话，保存在浏览器的 `localStorage` 中，因此下次打开应用时你的数据仍然在。

- 顶部的会话栏有一个列出所有会话的**下拉框**，以及当前会话的**名称**输入框。
- **+ New** 新建一个空白会话（日期为今天，无还款记录）。**Duplicate** 复制当前会话。**Delete** 删除当前会话（始终至少保留一个）。
- 编辑会在你停止输入片刻后**自动保存**；状态行会显示 “All changes saved · HH:MM:SS”。
- 启动时，应用会重新打开你上次使用的会话。首次使用者会得到一个预置的 “Example loan” 示例。

存储按浏览器、按设备隔离。如果 `localStorage` 不可用（隐私模式、受限浏览器），应用仍可运行，但数据只在本次访问期间保留，状态行也会提示这一点。

## 使用方法

1. **贷款设置**
   - **贷款金额（本金）**：发放的本金，单位人民币（CNY）。
   - **年利率**：基础利率，按百分数填写（例如 `6` 表示 6%）。
   - **起息日**：利息从这一天开始计算。
   - **到期日**：仅作参考。它本身不会改变利率；如需变更利率请使用罚息区间。
   - **计算截止日**：计算到哪一天为止。利息从起息日累计到这一天。
   - **计息基数**：`365`（民间借贷惯例）或 `360`（人民银行贷款计结息口径）。日利率 = 年利率 / 基数。

2. **罚息区间**（可选）
   - 点击 **+ Add window**。每个区间是一个左闭右开的日期范围 `[起, 止)`，在该范围内用它自己的年利率覆盖基础利率。
   - 典型用法：添加一个从到期日开始、使用较高罚息利率的区间，以模拟逾期期间。
   - 新建区间默认覆盖整个贷款期，利率为基础利率的 1.5 倍；可按需修改。
   - 如果区间重叠，以较晚添加的为准。

3. **还款**（可选）
   - 点击 **+ Add repayment**。输入日期和金额。每笔还款按以下顺序分摊（《中华人民共和国民法典》第 561 条）：
     1. **先冲利息**：先偿还已计提的未付利息。
     2. **再冲本金**：剩余部分用于偿还本金。
     3. **超额还款**：超过本金 + 利息的部分会显示为多付（贷方结余）。
   - 早于起息日的还款会被标记并忽略；晚于计算截止日的还款会被标记，暂不计入。

4. **查看结果**
   - **摘要**显示四个数字：本金余额、应计未付利息、已付利息合计、已付本金合计，以及截止日的应还总额。
   - **明细表**是按时间顺序排列的台账。每个计息行显示该区段的天数、利率和新增利息；每个还款行显示利息/本金的分摊以及随后的余额。

所有结果会随你输入实时重新计算，没有“计算”按钮。

### 快捷操作

- **Load example** 新建一个会话，内含一笔 100,000、利率 6% 的贷款，并在 2025-07-01 有一笔 50,000 的还款（不会覆盖你当前的会话）。
- **Run self-tests** 在浏览器中运行 5 个验收场景，并逐项显示通过/失败。

## 利息如何计算

时间轴在每个事件边界处被切分：起息日、每个罚息区间的边界、每个还款日，以及计算截止日。在每个区段内利率恒定，按单利计提：

```
segmentInterest = principalAtSegmentStart * (annualRate / dayBasis) * segmentDays
```

天数采用左闭右开：区段的起始日计入，结束日不计入。因此 2025-01-01 到 2025-12-31 为 364 天。未付利息会结转，但绝不计入本金（不计复利）。

全部运算以双精度浮点完成，仅在显示时四舍五入到两位小数。应还总额为 `round(本金 + 应计利息)`，因此它可能与你把两个已舍入的分项相加的结果相差一分钱。

## 说明与限制

- 仅支持单利。不计复利，也不提供等额本息 / 等额本金的摊还计划。
- 不强制法定上限（4 倍 LPR、24%/36% 红线）；应用按你输入的利率直接计算。
- 会话按浏览器存储在你的本机上；清除浏览器数据会一并删除。
- 不构成法律或财务建议。
